CONTENT部分

1.合理的title、desription、keywords：搜索权重减小。title不要出现两次且靠前；description不过分堆砌关键词，不同页面内容不同；keywords列举重要关键词。

<title>Document</title>
<meta name="description" content="这里填写描述内容"  />
<meta name="keywords" content="这里填写描述内容"  />
  1. 语义化的HTML代码，符合W3C规范：语义化代码让搜索引擎容易理解网页
  2. 重要内容不要用js输出：爬虫不会执行js获取内容
  3. 少用iframe：搜索引擎不会抓取iframe中的内容
  4. 非装饰性图片必须加alt
  5. 减少HTTP请求：合并文件、CSS精灵、inline Image
  6. 减少DNS查询：DNS查询完成之前浏览器不能从这个主机下载任何任何文件。方法：DNS缓存、
    将资源分布到恰当数量的主机名，平衡并行下载和DNS查询
  7. 避免重定向：多余的中间访问
  8. 使Ajax可缓存
  9. 非必须组件延迟加载
  10. 未来所需组件预加载
  11. 减少DOM元素数量
  12. 将资源放到不同的域下：浏览器同时从一个域下载资源的数目有限，增加域可以提高并行下载量
  13. 不要404

Server方面

  1. 使用CDN
  2. 添加Expires或者Cache-Control响应头
  3. 对组件使用Gzip压缩
  4. 配置ETag(实体标签(Entity Tag)的缩写。ETag一般不以明文形式相应给客户端。在资源的各个生命周期中，它都具有不同的值，用于标识出资源的状态。当资源发生变更时，如果其头信息中一个或者多个发生变化，或者消息实体发生变化，那么ETag也随之发生变化。)
  5. Flush the Buffer Early(尽早刷新)
  6. Ajax使用GET进行请求

Cookie方面

  1. 减小cookie大小
  2. 引入资源的域名不要包含cookie

css方面

  1. 将样式表放到页面顶部
  2. 不使用CSS表达式
  3. 压缩css
  4. 不使用IE的Filter

js方面

  1. 将脚本放置在body底部，不会因为下载和解析延长页面加载时间，使页面快速呈现
  2. 使用外部脚本，可维护性，可缓存，大势所趋
  3. 压缩javascript
  4. 删除不需要的脚本
  5. 减少DOM访问
  6. 合理设计事件监听器

图片方面

  1. 优化图片：根据实际颜色需要选择色深、压缩
  2. 优化css精灵
  3. 不要在HTML中拉伸图片
  4. 保证favicon.ico小并且可缓存

移动方面

  1. 保证组件小于25k
  2. 把Web 页面组件打包成一个多部分组成的文档。其目的是减少 HTTP 请求

html渲染过程

发表于 2018-05-13

HTML获取过程：

  1. 在浏览器地址栏输入URL

  2. 浏览器查看缓存，如果请求资源在缓存中并且新鲜，跳转到转码步骤

    i. 如果资源未缓存，发起新请求

    ii. 如果已缓存，检验是否足够新鲜，足够新鲜直接提供给客户端，否则与服务器进行验证。

    iii. 检验新鲜通常有两个HTTP头进行控制 Expires 和 Cache-Control ：
    HTTP1.0提供Expires，值为一个绝对时间表示缓存新鲜日期;HTTP1.1增加了Cache-Control: max-age=,值为以秒为单位的最大新鲜时间

  3. 浏览器解析URL获取协议，主机，端口，path

  4. 浏览器组装一个HTTP（GET）请求报文

  5. 浏览器获取主机ip地址，过程如下：
    i. 浏览器缓存

    ii. 本机缓存

    iii. hosts文件

    iv. 路由器缓存

    v. ISP DNS缓存

    vi. DNS递归查询（可能存在负载均衡导致每次IP不一样）

  6. 打开一个socket与目标IP地址，端口建立TCP链接，三次握手如下：

  7. i. 客户端发送一个TCP的SYN=1，Seq=X的包到服务器端口

    ii. 服务器发回SYN=1， ACK=X+1， Seq=Y的响应包

    iii. 客户端发送ACK=Y+1， Seq=Z

  8. TCP链接建立后发送HTTP请求
  9. 服务器接受请求并解析，将请求转发到服务程序，如虚拟主机使用HTTP Host头部判断请求的服务程
    序
  10. 服务器检查HTTP请求头是否包含缓存验证信息如果验证缓存新鲜，返回304等对应状态码
  11. 处理程序读取完整请求并准备HTTP响应，可能需要查询数据库等操作
  12. 服务器将响应报文通过TCP连接发送回浏览器

  13. 浏览器接收HTTP响应，然后根据情况选择关闭TCP连接或者保留重用，关闭TCP连接的四次握手如
    下：
    i. 主动方发送Fin=1， Ack=Z， Seq= X报文

    ii. 被动方发送ACK=X+1， Seq=Z报文

    iii. 被动方发送Fin=1， ACK=X， Seq=Y报文

    iv. 主动方发送ACK=Y， Seq=X报文

  14. 浏览器检查响应状态吗：是否为1XX，3XX， 4XX， 5XX，这些情况处理与2XX不同
  15. 如果资源可缓存，进行缓存
  16. 对响应进行解码（例如gzip压缩）
  17. 根据资源类型决定如何处理（假设资源为HTML文档）

SYN:TCP/IP建立连接时使用的握手信号

ACK:确认字符

Seq:seq是序列号

Fin:双方的数据传送完成

HTML渲染：

解析HTML文档，构件DOM树，下载资源，构造CSSOM树，执行js脚本，这些操作没有严格的先后顺序：

DOM构建：

  1. 构建DOM树：
    i. Tokenizing：根据HTML规范将字符流解析为标记

    ii. Lexing：词法分析将标记转换为对象并定义属性和规则

    iii. DOM construction：根据HTML标记关系将对象组成DOM树

  2. 解析过程中遇到图片、样式表、js文件，启动下载

  3. 构建CSSOM树：

    i. Tokenizing：字符流转换为标记流

    ii. Node：根据标记创建节点

    iii. CSSOM：节点创建CSSOM树

  4. 根据DOM树和CSSOM树构建渲染树:
    i. 从DOM树的根节点遍历所有可见节点，不可见节点包括：1） script , meta 这样本身不可见的标
    签。2)被css隐藏的节点，如 display: none

    ii. 对每一个可见节点，找到恰当的CSSOM规则并应用

    iii. 发布可视节点的内容和计算样式

js解析如下：

  1. 浏览器创建Document对象并解析HTML，将解析到的元素和文本节点添加到文档中，此时document.readystate为loading
  2. HTML解析器遇到没有async和defer的script时，将他们添加到文档中，然后执行行内或外部脚本。这些脚本会同步执行，并且在脚本下载和执行时解析器会暂停。这样就可以用document.write()把文本插入到输入流中。同步脚本经常简单定义函数和注册事件处理程序，他们可以遍历和操作script和他们之前的文档内容
  3. 当解析器遇到设置了async属性的script时，开始下载脚本并继续解析文档。脚本会在它下载完成
    后尽快执行，但是解析器不会停下来等它下载。异步脚本禁止使用document.write()，它们可以访问自己script和之前的文档元素
  4. 当文档完成解析，document.readState变成interactive
  5. 所有defer脚本会按照在文档出现的顺序执行，延迟脚本能访问完整文档树，禁止使用document.write()
  6. 浏览器在Document对象上触发DOMContentLoaded事件
  7. 此时文档完全解析完成，浏览器可能还在等待如图片等内容加载，等这些内容完成载入并且所有异步脚本完成载入和执行，document.readState变为complete,window触发load事件

    ending: 显示页面（HTML解析过程中会逐步显示页面）

DOM构建与解析顺序：

由于浏览器采用自上而下的方式解析，在遇到这两种元素（script、link）时都会阻塞浏览器的解析，直到外部资源加载并解析或执行完毕后才会继续向下解析html，外部样式不会阻塞后续脚本加载，但会阻塞后续脚本执行，直到外部样式加载并解析完毕。

load与DOMContentLoaded：

load:当一个资源及其依赖资源已完成加载时，将触发load事件。

DOMContentLoaded:HTML解析完毕

mongodb的使用

一.安装

快速讲解(windows安装)

1.下载mongodb安装包windows版，安装（可以自定义选择安装路径）,bin文件夹介绍：

2.在根目录下创建data文件夹，里面创建一个db文件夹用于存放数据。一个log文件夹用于存放日志

3.存放路径：mongod –dbpath f:\mongo\data\db

4.根目录创建mongod.congfig配置文件夹,输入：dbpath=f:\mongo\data\db
logpath=f:\mongo\data\log\mongod.log

5.配置：管理员启动cmd，输入mongod –config f:\mongo\mongod.config –install –serviceName “MongoDB”

6.启动：net start MongoDB

linux安装

ls进入，tar zxvf解压，mv移动,–dbpath,–logpath,–fork:后台执行

前端基本概念

1.HTML

HTML 指的是超文本标记语言 (Hyper Text Markup Language)。它不是一种编程语言，而是一种标记语言 (markup language)，可以用来描述网页。

2.XHTML

XHTML 指可扩展超文本标签语言（EXtensible HyperText Markup Language），比HTML更严格更纯净，作为一种XML 应用被重新定义的 HTML

3.XML

XML 指可扩展标记语言，被设计用来传输和存储数据

4.CSS

CSS 指层叠样式表 (Cascading Style Sheets)，定义如何显示 HTML 元素，可以嵌套在HTML中，也可以写成一个单独的文件分离出来。

5.JavaScript

轻量级的、可插入 HTML 页面的编程代码，是一种脚本语言。

6.jQuery

jQuery 是一个 JavaScript 函数库，包括：HTML元素选取、HTML 元素操作、CSS 操作、HTML 事件函数、JavaScript 特效和动画、HTML DOM、遍历和修改、AJAX、Utilities。

7.AJAX

AJAX 是一种用于创建快速动态网页的技术。
通过在后台与服务器进行少量数据交换，AJAX 可以使网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新

8.JSON

JSON是一种键值对结构的数据，例如：

{"name":"Bill Gates","street":"Fifth Avenue New York 666","age":56,"phone":"555 1234567"}

9.PHP

PHP 是一种被广泛使用的开源脚本语言，可以用来：

  • 生成动态页面内容
  • 创建、打开、读取、写入、关闭服务器上的文件
  • 收集表单数据
  • 发送和接收 cookies
  • 添加、删除、修改您的数据库中的数据
  • 限制用户访问您的网站上的一些页面
  • 加密数据

10.JAVA

Java是Java面向对象程序设计语言和Java平台的总称，可运行于多个平台，如Windows, Mac OS，及其他多种UNIX版本的系统

11.数据库

数据库（Database）是按照数据结构来组织、存储和管理数据的仓库，每个数据库都有一个或多个不同的API用于创建，访问，管理，搜索和复制所保存的数据。

12.MySQL

Mysql是最流行的关系型数据库管理系统，用来存储、管理、操作数据。

13.SQL

SQL指结构化查询语言，全称是 Structured Query Language，可以用来访问和处理数据库。

14.JSP

JSP(Java Server Pages)是运行在服务端的语言，可以响应客户端请求，而动态生成 HTML、XML或其他格式文档的Web网页的技术标准。
